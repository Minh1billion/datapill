DataPrep CLI - dp

A command-line tool for ingesting, profiling, preprocessing, and exporting data across PostgreSQL, MySQL, S3, Kafka, REST APIs, and local files.

Python ≥ 3.11 Polars · PyArrow asyncio-native entry: dp

Installation

Install the package in editable mode from the project root. The dp entry point is registered automatically via pyproject.toml.

bash
# Install with dev dependencies
pip install -e ".[dev]"

# Verify
dp --help

Quick Start

The minimal path from a CSV file to a profiled, preprocessed, exported dataset.

  1. Ingest a local CSV file into the artifact store.
  2. Profile the ingested dataset to understand its structure.
  3. Apply preprocessing transformations.
  4. Export the result to Parquet or write back to a database.
bash
dp ingest --source local_file --path data.csv
# -> Artifact: abc123_ingest_output

dp profile --input abc123_ingest_output

dp preprocess --input abc123_ingest_output --pipeline pipeline.json
# -> Output: def456_preprocess_output

dp export --input def456_preprocess_output --format parquet --out-path output/result.parquet

PostgreSQL

bash
docker compose -f docker-compose.test.yml exec -T postgres \
  psql -U testuser -d testdb \
  -c "
CREATE TABLE IF NOT EXISTS orders (
    id SERIAL PRIMARY KEY,
    product_name VARCHAR(100),
    category VARCHAR(50),
    price NUMERIC(10,2),
    quantity INTEGER,
    status VARCHAR(20),
    created_at TIMESTAMP DEFAULT NOW()
);
INSERT INTO orders (product_name, category, price, quantity, status)
SELECT
    'Product_' || i,
    (ARRAY['Electronics','Clothing','Food','Books'])[1 + (i % 4)],
    round((random() * 1000)::numeric, 2),
    (random() * 100)::int,
    (ARRAY['pending','shipped','delivered','cancelled'])[1 + (i % 4)]
FROM generate_series(1, 50000) i;
"

Kafka - Produce test messages

bash
# Create topic first (use // prefix on Git Bash to prevent path conversion)
docker compose -f docker-compose.test.yml exec -T kafka \
  //opt/kafka/bin/kafka-topics.sh \
  --bootstrap-server localhost:9092 \
  --create --topic test-topic \
  --partitions 1 --replication-factor 1

# Produce 100 JSON messages
python - << 'EOF'
import asyncio, json
from aiokafka import AIOKafkaProducer

async def produce():
    p = AIOKafkaProducer(bootstrap_servers="localhost:9092")
    await p.start()
    for i in range(100):
        msg = json.dumps({"id": i, "product_name": f"Product_{i}",
            "category": ["Electronics","Clothing","Food","Books"][i%4],
            "price": round(i*10.5,2), "quantity": i%50}).encode()
        await p.send("test-topic", msg)
    await p.stop()
    print("Produced 100 messages")

asyncio.run(produce())
EOF

dp ingest

Read data from a connector and persist it as a Parquet artifact. Returns an artifact ID used by downstream commands.

OptionDefaultDescription
--source, -srequiredConnector type
--config, -c-Path to connector JSON config
--path-Local file path (local_file)
--table-Table name (postgresql | mysql)
--url-S3 URL s3://bucket/key
--topic-Kafka topic name
--endpoint-REST endpoint path
--out, -osrc/dataprep/artifactsArtifact store directory
--limit, -n-Max rows to read
--batch-size50 000Rows per streaming batch
--max-records-Max messages consumed (kafka)
bash
# Local CSV
dp ingest --source local_file --path tests/fixtures/data.csv

# Local CSV - large file with row limit
dp ingest --source local_file --path tests/fixtures/data_1M.csv \
  --batch-size 100000 --limit 50000

# PostgreSQL
dp ingest --source postgresql --config tests/fixtures/pg_config.json --table orders

# MySQL
dp ingest --source mysql --config tests/fixtures/mysql_config.json --table products

# S3
dp ingest --source s3 --config tests/fixtures/s3_config.json \
  --url s3://testbucket/input/data.csv

# REST API (no leading slash on Git Bash)
dp ingest --source rest --config tests/fixtures/rest_config.json --endpoint posts

# Kafka - consume up to 100 messages
dp ingest --source kafka --config tests/fixtures/kafka_config.json \
  --topic test-topic --max-records 100

dp profile

Compute descriptive statistics, null rates, distinct counts, correlations, and quality warnings for a dataset.

OptionDefaultDescription
--input, -irequiredArtifact ID or .parquet file path
--mode, -mfullfull | summary
--sample-strategynonenone | random | reservoir
--sample-size100 000Rows to sample when strategy ≠ none
--correlationpearsonpearson | spearman | none
--out, -osrc/dataprep/artifactsArtifact store directory
bash
# Full profile
dp profile --input 72e0548a_ingest_output

# Full profile with random sampling (large datasets)
dp profile --input 72e0548a_ingest_output \
  --mode full \
  --sample-strategy random \
  --sample-size 10000 \
  --correlation pearson

# Profile directly from a parquet file
dp profile --input output/result.parquet --mode summary

dp preprocess

Apply a sequence of transformation steps defined in a JSON pipeline file.

OptionDefaultDescription
--input, -irequiredArtifact ID or .parquet file path
--pipeline, -prequiredPath to pipeline JSON config
--dry-runfalseRun on first 1 000 rows, skip artifact save
--checkpointfalseSave a Parquet checkpoint after each step
--out, -osrc/dataprep/artifactsArtifact store directory

Pipeline JSON format

json - tests/fixtures/preprocess_pipeline.json
{
  "steps": [
    {"type": "impute_mean",    "scope": {"columns": ["price", "quantity"]}},
    {"type": "clip_iqr",       "scope": {"columns": ["price"]}},
    {"type": "standard_scaler", "scope": {"columns": ["price", "quantity"]}}
  ]
}
bash
# Dry run first
dp preprocess \
  --input    72e0548a_ingest_output \
  --pipeline tests/fixtures/preprocess_pipeline.json \
  --dry-run

# Full run with checkpoint
dp preprocess \
  --input    72e0548a_ingest_output \
  --pipeline tests/fixtures/preprocess_pipeline.json \
  --checkpoint

dp export

Write a dataset to a local file or push it back to a database or S3.

OptionDefaultDescription
--input, -irequiredArtifact ID or .parquet path
--format, -frequiredcsv | parquet | json | jsonl | excel
--out-path-Destination file path (file export)
--connector, -c-Write-back connector config JSON
--write-modereplacereplace | append | upsert
--primary-keys-Comma-separated keys for upsert
--compression-snappy | zstd | gzip (parquet only)
--dry-runfalsePrint first 10 rows, skip write
bash
# Dry run - preview 10 rows
dp export --input c4ccb55d_preprocess_output --format parquet \
  --out-path output/result.parquet --dry-run

# Export to Parquet
dp export --input c4ccb55d_preprocess_output --format parquet \
  --out-path output/result.parquet

# Export to CSV
dp export --input c4ccb55d_preprocess_output --format csv \
  --out-path output/result.csv

# Parquet with zstd compression
dp export --input c4ccb55d_preprocess_output --format parquet \
  --out-path output/result.zstd.parquet --compression zstd

# Write back to PostgreSQL - replace
dp export --input c4ccb55d_preprocess_output --format parquet \
  --connector tests/fixtures/pg_write_config.json --write-mode replace

# Write back to PostgreSQL - upsert
dp export --input c4ccb55d_preprocess_output --format parquet \
  --connector tests/fixtures/pg_write_config.json \
  --write-mode upsert --primary-keys id

# Write back to MySQL - replace
dp export --input d7859e9e_ingest_output --format parquet \
  --connector tests/fixtures/mysql_write_config.json --write-mode replace

# Write back to S3
dp export --input c4ccb55d_preprocess_output --format parquet \
  --connector tests/fixtures/s3_write_config.json
⚠ append + primary key constraint: Using --write-mode append on a table with a primary key will fail if duplicate IDs exist. Use upsert with --primary-keys instead, or drop the constraint before appending.

dp run

Execute a full ingest + profile pipeline in a single command from a JSON config file.

Config file format

json - tests/fixtures/run_config_pg.json
{
  "source": "postgresql",
  "connector": {
    "host": "localhost", "port": 5433,
    "database": "testdb", "user": "testuser", "password": "testpass"
  },
  "query":   { "table": "orders" },
  "ingest":  { "batch_size": 10000 },
  "profile": {
    "mode": "full", "sample_strategy": "random",
    "sample_size": 10000, "correlation": "pearson"
  }
}

dp run tests/fixtures/run_config_pg.json

# Custom artifact output directory
dp run tests/fixtures/run_config_pg.json --out /tmp/dp_artifacts

Walkthrough - PostgreSQL end-to-end

bash
# 1. Verify connection
dp connector test   --source postgresql --config tests/fixtures/pg_config.json
dp connector schema --source postgresql --config tests/fixtures/pg_config.json --table orders

# 2. Ingest

# -> Artifact: 72e0548a_ingest_output

# 3. Profile
dp profile --input 72e0548a_ingest_output \
  --sample-strategy random --sample-size 10000

# 4. Preprocess
dp preprocess \
  --input    72e0548a_ingest_output \
  --pipeline tests/fixtures/preprocess_pipeline.json \
  --checkpoint
# -> Output: c4ccb55d_preprocess_output

# 5a. Export to Parquet
dp export --input c4ccb55d_preprocess_output --format parquet \
  --out-path output/result.parquet

# 5b. Or write back to PostgreSQL
dp export --input c4ccb55d_preprocess_output --format parquet \
  --connector tests/fixtures/pg_write_config.json \
  --write-mode upsert --primary-keys id

Write Modes

ModeBehaviourRequires
replaceTRUNCATE the target table, then INSERT all rows.Table must exist.
appendINSERT all rows without touching existing data.Table must exist. Fails if a unique constraint is violated.
upsertINSERT … ON CONFLICT DO UPDATE for each row.Table must have a primary key or unique constraint on --primary-keys columns.
✓When in doubt, use upsert with a primary key. It is idempotent and safe to re-run.

Pipeline Steps

Step typeDescriptionTypical columns
impute_meanFill null values with the column mean.Numeric
impute_medianFill null values with the column median.Numeric
impute_modeFill null values with the most frequent value.Any
clip_iqrClip outliers to [Q1 − 1.5×IQR, Q3 + 1.5×IQR].Numeric
standard_scalerStandardise to zero mean and unit variance.Numeric
min_max_scalerScale values to the [0, 1] range.Numeric
log_transformApply log1p transformation.Non-negative numeric
drop_duplicatesRemove exact duplicate rows.All (or subset)
drop_nullsDrop rows where specified columns are null.Any
